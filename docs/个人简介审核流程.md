# 个人简介审核流程（精简版）

## 1. 目标
- 覆盖资料更新审核（`profile_stage=update`）与被举报复审（`review_stage=reported`）。
- 保持统一输出：`ALLOW | REJECT | ESCALATE` + 可验证证据。
- 支持多字段文本与头像图像联合审核，并落地字段级最小处置。

## 2. 固定口径
### 2.1 提示词
- 仅保留：
  - `MODERATION_TEXT`
  - `MODERATION_VISION`
  - `MODERATION_JUDGE`

### 2.2 硬规则
- 举报信号仅用于触发分流，不作为违规证据。
- 任意 `REJECT` 必须有可验证证据（字段原文、OCR、图片编号）。
- 证据不足、冲突未消解时必须 `ESCALATE`。

## 3. 输入与输出
### 3.1 输入
- `user_id`、`profile_id`（可选）、`updated_at`
- `profile_fields`：
  - `username`（必填）
  - `bio`、`location`、`website`（可空）
  - `avatar`（`avatar_url` / `avatar_image_bytes` / `avatar_hash` / 元信息）
- `old_profile_fields`（可选，用于回滚）
- `lang`、`user_context`（可选）
- `report_context`（复审场景可选，含 `content_snapshot_id`）
- `policy_version`
- `profile_stage` 或 `review_stage`（二选一）

### 3.2 输出
- `risk_score`、`decision_suggestion`、`labels[]`、`severity`
- `evidence[]`（可定位、可验证）
- `uncertainty`、`notes`

## 4. 路由规则
- 文本字段为主：`TextAudit`。
- 头像为主：`VisionAudit + TextAudit` 并行，`Judge` 融合。
- 复审场景：不新增模型链路，仅差异化前置与阈值策略。

## 5. 三阶段流水线
### Step 0：预处理（LLM 前）
- 文本规范化并保留原文映射，防止证据回溯失败。
- 业务字段校验优先（长度、格式、URL 协议）。
- 反垃圾前置：短窗频繁改资料、模板引流、规避写法检测。
- 头像预处理：下载/解码/hash，保障复审可重放。

### Step 1：并行取证
- `TextAudit`：基于 `profile_fields`（保留字段边界）。
- `VisionAudit`：针对头像输出风险、标签、OCR 与证据。
- 要求：证据必须可定位到字段原文/OCR/图片编号。

### Step 2：聚合初判
- `profile_risk_max = max(text_risk, image_risk)`。
- 硬门槛先行，冲突与证据校验优先。
- 支持字段级处置建议（仅问题字段回滚/清空）。

### Step 3：升级仲裁
- 触发条件：
  - 证据不足但风险偏高
  - 文本与视觉结论冲突
  - 复审场景与历史结论冲突
- 升级后仍证据不足时必须 `ESCALATE`。

## 6. 决策规则
### 6.1 阈值生效顺序
1) 先取 `thresholds.default` 或 `by_review_stage[review_stage].default`。
2) 再按 `labels[]` 应用 `by_label`。
3) 多标签命中取更严格阈值（`min`）。

### 6.2 硬门槛
- `REJECT` 前提：至少 1 条可验证 `evidence`。
- 以下情况必须 `ESCALATE`：
  - 证据为空或不可验证
  - 依赖 OCR 但 OCR 证据质量不足
  - 文本与视觉结论冲突且无法消解
  - 规则校验失败

## 7. 资料更新审核（profile_stage=update）
- 默认路径：`TextAudit -> 策略层 -> 必要时 Judge`。
- 带头像路径：`VisionAudit + TextAudit -> Judge -> 策略层`。
- 动作映射：
  - `ALLOW`：本次资料变更生效
  - `REJECT`：拒绝本次变更，可字段级回滚/清空，保持旧版本
  - `ESCALATE`：进入人工队列，可临时隐藏敏感字段或维持旧版本

## 8. 被举报复审（review_stage=reported）
- 复审前置：去重、限流、聚合信号、快照冻结（`content_snapshot_id`）。
- 策略差异：更偏召回、证据更严格、冲突更强升级。
- 处置差异：优先字段级最小处置（如清空 `bio/website`、撤下头像）。

## 9. 最小可落地配置
- `thresholds.default`：`T_allow=0.20`，`T_reject=0.80`
- `by_review_stage.reported`：`T_allow=0.15`，`T_reject=0.75`
- `by_review_stage.appeal`：`T_allow=0.25`，`T_reject=0.85`
- `escalate_rules.require_evidence=true`
- `escalate_rules.prefer_field_level_actions=true`
- `field_validation_limits`：用户名/简介/地区/网站长度与格式约束
- `anti_spam_rules`：资料修改频控与相似度规则
