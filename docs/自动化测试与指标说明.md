# 自动化测试与指标说明

## 一键运行

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\test-all.ps1
```

可选参数：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\test-all.ps1 -SkipSecurity
powershell -ExecutionPolicy Bypass -File .\scripts\test-all.ps1 -SkipPerf
```

产物默认输出到 `test-reports/`，并生成 `test-reports/SUMMARY.md` 作为总览入口。

## 功能测试（回归）

- 后端：JUnit 5 + Spring Boot Test（含 Security Test），报告：`build/reports/tests/test/`、`build/reports/tests/integrationTest/`
- 前端：Vitest + Testing Library，报告：`my-vite-app/test-reports/vitest-junit.xml`

推荐指标：
- 用例数、通过率、失败用例列表（按模块/接口分类）
- 回归覆盖的关键业务路径（登录、权限、发帖/评论、检索/问答等）

## 覆盖率

- 后端：JaCoCo HTML/XML
  - 单元：`build/reports/jacoco/test/html/`
  - 集成：`build/reports/jacocoIntegrationTestReport/html/`
- 前端：Vitest v8 覆盖率：`my-vite-app/test-reports/vitest-coverage/`

推荐指标：
- 行覆盖率、分支覆盖率、方法覆盖率（后端）
- 语句/分支/函数/行覆盖率（前端）
- 关键模块覆盖率对比（核心服务层 > 辅助工具层 > UI）

## 高并发/性能测试

- 工具：Apache JMeter（非 GUI 运行，生成 HTML 报告）
- 计划文件：`perf/jmeter/EnterpriseRagCommunity_basic_load.jmx`
- 运行脚本：`perf/jmeter/run-jmeter.ps1`
- 报告产物：`perf/jmeter/results/<时间戳>/html-report/`

推荐指标（毕业设计常用口径）：
- 吞吐量（Throughput / TPS）
- 平均响应时间、P90/P95/P99
- 错误率（Error %）
- 并发数（Threads / Users）

## 服务端资源指标（压测配套）

- 通过 `application-perf.properties` 启用 Prometheus 指标端点（默认仅本机 127.0.0.1）
  - 指标端口：`http://127.0.0.1:8098/actuator/prometheus`
- 推荐在压测期间用 Prometheus+Grafana 或其他采集方式记录：
  - CPU、内存、GC、线程数
  - HTTP 请求耗时分布（如果有相应 Micrometer 指标）

## 安全测试

### SCA（依赖漏洞）
- 工具：OWASP Dependency-Check（Gradle 任务：`dependencyCheckAnalyze`）
再次运行：
```powershell
.\gradlew.bat dependencyCheckAnalyze --no-daemon --rerun
```
需要更新/清理 NVD 数据库（可选）
```powershell
# 更新 NVD 数据
.\gradlew.bat dependencyCheckUpdate --no-daemon

# 清理本地 NVD 缓存（再跑一次分析会重新下载/构建数据）
.\gradlew.bat dependencyCheckPurge --no-daemon
.\gradlew.bat dependencyCheckAnalyze --no-daemon --rerun
```

- 报告：`build/reports/dependency-check/dependency-check-report.html`、`dependency-check-report.json`

### DAST（动态扫描）
- 工具：OWASP ZAP（Docker 方式运行基线扫描）
- 脚本：`security/zap-baseline-docker.ps1`
- 报告：`security/reports/zap-baseline.html`

### 前端依赖审计
- 工具：npm audit（脚本：`security/npm-audit.ps1`）
- 产物：`security/reports/npm-audit.json`

推荐指标：
- 漏洞数（按严重等级 Critical/High/Medium/Low）
- 漏洞类型（可映射 OWASP Top 10）
- 依赖漏洞清单（CVE/组件版本/修复建议）
