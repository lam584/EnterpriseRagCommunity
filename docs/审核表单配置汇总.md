# 审核表单配置汇总（参数手册）

## 1. 文档定位
- 本文是“配置真源”，只保留可执行参数与约束，不重复流程叙述。
- 适用对象：帖子、评论、个人简介三类审核场景。

## 2. 全局口径
### 2.1 提示词
- 仅 3 个 `prompt_code`：
  - `MODERATION_TEXT`
  - `MODERATION_VISION`
  - `MODERATION_JUDGE`


### 2.2 全局硬约束
- 模型仅输出 JSON。
- `decision_suggestion` 仅允许：`ALLOW` / `REJECT` / `ESCALATE`。
- `labels[]` 必须来自 `label_taxonomy.allowed_labels`。
- `risk_score`、`uncertainty` 均为 `0~1`。
- 任意 `REJECT` 必须附可验证证据；否则必须 `ESCALATE`。
- 举报理由不作为违规证据，只用于触发与分流。

### 2.3 证据约束
- 证据必须可定位到原文/OCR/图片/字段。
- `quote` 若存在必须是输入原样子串。
- 推荐使用上下文锚点（`before_context` / `after_context`）提高可验性。

## 3. 模型默认参数
| 参数 | 默认值 | 说明 |
|---|---:|---|
| temperature | 0.1 | 低随机性，提升稳定性 |
| top_p | 1.0 | 与低 temperature 配合使用 |
| TextAudit.max_output_tokens | 1200 | 结构化输出上限 |
| VisionAudit.max_output_tokens | 1000 | 结构化输出上限 |
| Judge.max_output_tokens | 1500 | 融合裁决输出上限 |
| TextAudit.reasoning_mode | off | 初审以稳定为主 |
| VisionAudit.reasoning_mode | off | 初审以稳定为主 |
| Judge.reasoning_mode | off/auto | 初判 off，升级仲裁 auto/on |

## 4. 三类任务输入输出契约
## 4.1 TextAudit
### 输入最小集
- `policy_version`
- `label_taxonomy`
- 业务主键（`post_id` / `comment_id` / `user_id`）
- 文本内容（正文或字段文本）
- 可选 `related_ocr`、`user_context`、`review_stage`

### 输出最小集
- `risk_score`
- `decision_suggestion`
- `labels[]`
- `severity`
- `evidence[]`
- `uncertainty`
- `notes`

## 4.2 VisionAudit
### 输入最小集
- `policy_version`
- `label_taxonomy`
- 业务主键
- `image_id`
- 图片数据（url 或 bytes）

### 输出最小集
- `image_risk_score`
- `decision_suggestion`
- `image_labels[]`
- `severity`
- `ocr_text`、`ocr_confidence`
- `evidence[]`
- `uncertainty`
- `notes`

## 4.3 Judge
### 输入最小集
- `policy_version`
- `label_taxonomy`
- `thresholds`
- `escalate_rules`
- `text_results[]`
- `image_results[]`
- 可选 `evidence_book`、`global_memory`

### 输出最小集
- `risk_score`
- `decision_suggestion`
- `labels[]`
- `severity`
- `evidence[]`
- `uncertainty`
- `notes`

## 5. 决策阈值参数
## 5.1 默认阈值
| 字段 | 默认值 | 说明 |
|---|---:|---|
| thresholds.default.T_allow | 0.20 | 低风险直通 |
| thresholds.default.T_reject | 0.80 | 高风险拒绝候选 |

## 5.2 场景覆盖阈值
| 字段 | 默认值 | 说明 |
|---|---:|---|
| thresholds.by_review_stage.reported.T_allow | 0.15 | 复审更偏召回 |
| thresholds.by_review_stage.reported.T_reject | 0.75 | 复审降低漏放 |
| thresholds.by_review_stage.appeal.T_allow | 0.25 | 申诉更偏精确 |
| thresholds.by_review_stage.appeal.T_reject | 0.85 | 申诉降低误伤 |

## 5.3 类目阈值覆盖
- `thresholds.by_label.<label>.T_allow/T_reject` 按类目配置。
- 多标签命中时取更严格阈值（`min`）。

## 5.4 阈值生效顺序
1) 先取 `default` 或 `by_review_stage` 覆盖值。
2) 再应用 `by_label` 覆盖。
3) 阈值只决定区间，不替代硬门槛。

## 6. 升级与硬门槛参数
| 字段 | 默认值 | 作用 |
|---|---:|---|
| escalate_rules.require_evidence | true | 证据不可验时禁止 REJECT |
| escalate_rules.escalate_on_label_conflict | true | 多源标签冲突强制升级 |
| escalate_rules.escalate_on_ocr_weak_evidence | true | 依赖 OCR 且证据弱时升级 |
| escalate_rules.escalate_on_context_missing | true | 上下文缺失时升级 |

## 7. 举报触发复审参数
## 7.1 聚合窗口
| 字段 | 默认值 | 说明 |
|---|---:|---|
| review_trigger.window_minutes | 10 | 举报速度统计窗口 |

## 7.2 分流阈值
| 队列 | 触发条件（默认） |
|---|---|
| light | `report_count_unique >= 3` |
| standard | `report_count_unique >= 5` 或 `report_velocity >= 5` 或 `reporter_trust_agg >= 0.7` |
| urgent | `report_count_unique >= 20` 或 `report_velocity >= 20` 或（`reporter_trust_agg >= 0.85` 且命中高危理由） |

## 7.3 复审通用要求
- 必须生成并使用 `content_snapshot_id`。
- 复审判定只看内容证据，不看举报文案本身。

## 8. 反垃圾前置参数

| anti_spam.comment.similarity_threshold | 0.90 |
| anti_spam.comment.max_similar_count_per_10min | 3 |

## 8.2 个人简介

|---|---:|
